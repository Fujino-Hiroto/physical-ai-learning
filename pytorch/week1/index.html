1 PyTorch とは何か ▶
概念

NumPy の GPU 版 + 自動微分エンジン + ニューラルネットフレームワーク。 この 3 つが 1 つのライブラリに統合されているのが PyTorch。

何ができるか

  • テンソル計算：行列演算を CPU でも GPU でも同じコードで実行
  • 自動微分：勾配を手で計算しなくてよい（backward() 一発）
  • ネットワーク定義・学習：層を積んで重みを最適化
  • 既存モデル活用：学習済み YOLO・ResNet を数行でロード

コード例：CPU でも GPU でも同じ書き方

import torch

# デバイスを自動選択（Mac は "mps"、NVIDIA は "cuda"）
device = "cuda" if torch.cuda.is_available() else "cpu"

x = torch.tensor([[1., 2.], [3., 4.]], device=device)
w = torch.randn(2, 2, device=device)
y = x @ w          # 行列積。GPU 上でそのまま実行される
print(y.shape)     # torch.Size([2, 2])
🔬 研究への接点

RealSense D435 から得られる RGB 画像や点群（point cloud）は、すべて テンソルとして PyTorch に入力できる。 SLAM の特徴抽出や深度補完を学習ベースで行う際、torch の GPU テンソルがそのまま処理パイプラインの中核になる。

2 テンソルとは何か ▶
概念

多次元配列。スカラー・ベクトル・行列をすべて含む上位概念で、 画像・音声・センサデータを統一した形式で扱える。

覚えておきたい属性

属性意味例
.shape各次元のサイズ[8, 3, 480, 640]
.dtype数値型float32 / uint8
.device置き場所cpu / cuda
import torch

img = torch.rand(3, 480, 640)      # RGB画像 1枚: [C, H, W]
batch = img.unsqueeze(0)           # 先頭に N 軸を追加 → [1, 3, 480, 640]
print(batch.shape)                 # torch.Size([1, 3, 480, 640])

gray = img.mean(dim=0)             # チャンネル方向に平均 → [480, 640]（2Dテンソル）
print(gray.shape)                  # torch.Size([480, 640])
🔬 研究への接点

D435 の深度マップは 2D テンソル（高さ×幅）、RGB 画像は 3D テンソル（C×H×W）。 両者を [4, H, W] の RGB-D テンソルとして結合すれば、点群生成や SLAM の入力を 1 つのテンソルで統一的に扱える。

3 自動微分（Autograd） ▶
概念

計算の手順を計算グラフとして自動で記録し、誤差逆伝播（backpropagation）を自動計算する仕組み。 手で微分式を書かなくても、loss.backward() の 1 行で全パラメータの勾配が求まる。

直感：勾配とは「どっちにどれだけ動かすと損失が減るか」

例として \( y = a x + b \) を考える。\(a, b\) を少し動かしたとき \(y\) がどれだけ変わるかが「勾配」。 PyTorch は 連鎖律（chain rule）をグラフに沿って自動適用する：

\[ \frac{\partial L}{\partial a} = \frac{\partial L}{\partial y}\cdot\frac{\partial y}{\partial a} = \frac{\partial L}{\partial y}\cdot x \]

import torch

a = torch.tensor(2.0, requires_grad=True)   # 勾配を追跡する
x = torch.tensor(3.0)
b = torch.tensor(1.0, requires_grad=True)

y = a * x + b          # 順伝播：計算グラフが自動構築される
loss = (y - 10) ** 2   # 目標値10との誤差

loss.backward()        # ★ここで全パラメータの勾配を自動計算

print(a.grad)          # dL/da = 2(y-10)*x = 2(7-10)*3 = -18
print(b.grad)          # dL/db = 2(y-10)   = -6
🔬 研究への接点

D435 の深度には距離依存のノイズが乗る。ノイズモデル \( \sigma(d)=\alpha d^2+\beta \) のパラメータ \(\alpha,\beta\) を 観測データにフィットさせる際、autograd で勾配を自動計算すれば、最尤推定や最小二乗フィッティングを数十行で実装できる。

4 ニューラルネットワークの構造（nn.Module） ▶
概念

nn.Module を継承してネットワークを定義する。 全結合・CNN・RNN・Transformer まで、すべて同じ枠組みで書ける。

ルールは 2 つだけ

  • __init__ で使う層を宣言する（重みはここで自動生成）
  • forward でデータの流れを書く（順伝播の定義）
import torch.nn as nn

class MyNet(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc1 = nn.Linear(784, 128)   # 入力784 → 隠れ128
        self.fc2 = nn.Linear(128, 10)    # 隠れ128 → 出力10クラス
        self.act = nn.ReLU()

    def forward(self, x):
        x = self.act(self.fc1(x))        # 全結合 → 活性化
        return self.fc2(x)               # 最終出力（クラスごとのスコア）

model = MyNet()
print(model)                              # 構造がそのまま表示される
print(sum(p.numel() for p in model.parameters()))  # 総パラメータ数
🔬 研究への接点

LeRobot のポリシーネットワークも、画像エンコーダや行動デコーダがすべて nn.Module で定義されている。 この枠組みを理解しておけば、既存ロボット制御モデルの中身を読み解き、UR5e 向けに出力層を差し替える改造も容易になる。

5 学習ループの全体像 ▶
概念

深層学習の「学習」は、次の 5 ステップの繰り返し： データ読み込み → 順伝播 → 損失計算 → 逆伝播 → パラメータ更新。 どんなタスクでもこの骨格は変わらない。

import torch.nn as nn, torch.optim as optim

model = MyNet()
loss_fn = nn.CrossEntropyLoss()
optimizer = optim.Adam(model.parameters(), lr=1e-3)

for epoch in range(5):
    for x, y in train_loader:            # ① データ取得（バッチ単位）
        optimizer.zero_grad()            #   前回の勾配をリセット（重要）
        pred = model(x)                  # ② 順伝播
        loss = loss_fn(pred, y)          # ③ 損失計算
        loss.backward()                  # ④ 逆伝播（勾配を計算）
        optimizer.step()                 # ⑤ パラメータ更新
    print(f"epoch {epoch}: loss={loss.item():.4f}")

⚠️ optimizer.zero_grad() を忘れると勾配が累積して学習が壊れる。ループ先頭での呼び出しが定石。

🔬 研究への接点

強化学習（RL）の学習ループも、報酬から損失を作る点が違うだけで 基本構造はまったく同じ。 この 5 ステップを体に入れておけば、模倣学習・方策勾配・SLAM の学習ベース手法へスムーズに移行できる。

6 損失関数と最適化 ▶
概念

損失関数でモデルの「悪さ」を 1 つの数値に定量化し、 最適化器がその数値を小さくする方向へパラメータを更新する。

代表的な損失関数

損失用途イメージ
MSELoss回帰（連続値の予測）予測と正解の差の二乗平均
CrossEntropyLoss分類（クラス予測）正解クラスの確率を上げる
L1Loss回帰（外れ値に強い）差の絶対値平均

MSE の定義は \(\; L=\dfrac{1}{N}\sum_{i}(\hat{y}_i-y_i)^2\;\)。 「予測 \(\hat{y}\) と正解 \(y\) の食い違い」を測る、最も直感的な損失。

最適化器：SGD と Adam

  • SGD：勾配の向きに一定歩幅で進むシンプルな方法
  • Adam：過去の勾配を加味して歩幅を自動調整。まず Adamで困らない
import torch, torch.nn as nn

# y = 2x + 1 を当てる単回帰でフィッティングの雰囲気をつかむ
x = torch.randn(100, 1)
y = 2 * x + 1 + 0.1 * torch.randn(100, 1)

model = nn.Linear(1, 1)
loss_fn = nn.MSELoss()                                # 最小二乗 = MSE 最小化
optimizer = torch.optim.Adam(model.parameters(), lr=0.05)

for step in range(200):
    optimizer.zero_grad()
    loss = loss_fn(model(x), y)
    loss.backward()
    optimizer.step()

w, b = model.weight.item(), model.bias.item()
print(f"学習結果: y = {w:.2f}x + {b:.2f}")            # ≈ y = 2.00x + 1.00
🔬 研究への接点

D435 のノイズモデルを観測データにフィットさせる作業は、まさに 最小二乗法 ＝ MSE 最小化。 上のコードの nn.Linear を自前のノイズモデル関数に置き換えれば、同じ枠組みでキャリブレーションができる。

7 既存モデルの活用（torchvision・YOLO） ▶
概念

学習済みの重み（pretrained weights）をロードして、推論やファインチューニングに使う。 大規模データで鍛えた特徴抽出能力を「借りる」ので、少ないデータ・短い学習で高精度を出せる。

例A：torchvision の学習済み ResNet で画像分類

import torch
from torchvision import models, transforms

model = models.resnet50(weights="IMAGENET1K_V2")   # 学習済み重みをDL
model.eval()                                        # 推論モードに切替

preprocess = transforms.Compose([
    transforms.Resize(256), transforms.CenterCrop(224),
    transforms.ToTensor(),
    transforms.Normalize([0.485, 0.456, 0.406], [0.229, 0.224, 0.225]),
])

x = preprocess(img).unsqueeze(0)                    # [1, 3, 224, 224]
with torch.no_grad():                               # 勾配不要 → 高速・省メモリ
    logits = model(x)
print(logits.argmax(1))                             # 予測クラスID

例B：YOLO で物体検出（最小実装）

# pip install ultralytics
from ultralytics import YOLO

model = YOLO("yolov8n.pt")                  # 軽量な学習済みモデル
results = model("greenhouse.jpg")           # 推論はこの1行
results[0].show()                           # 検出枠を描画して表示

# 自分のデータでファインチューニング（出力層を温室データに適応）
model.train(data="tomato.yaml", epochs=50, imgsz=640)
🔬 研究への接点

温室内の植物・果実検出は、YOLO のファインチューニングが第一候補。 D435 の RGB で果実を検出し、対応する深度から 3D 位置を復元すれば、UR5e による収穫アームの coarse-to-fine プランニング（おおまかに接近 → 精密把持）の「対象位置」入力として直接使える。
